# Prompt para Agente IA (Go Developer) — Proyecto `wiki-llm`

## Encabezado estándar
Genera una intervención técnica clara y ejecutable para un agente IA orientado a desarrollo en Go. El agente debe actuar con criterio de arquitectura limpia, simplicidad y preparación para crecimiento iterativo.

---

## Estado inicial del repositorio (CRITICO)

El proyecto está **vacío** y se debe construir **desde cero**.

- No existen archivos previos a conservar.
- Debes inicializar el repositorio como base funcional mínima.
- No asumas código preexistente.

---

## Objetivo de la intervención

Inicializar la estructura base del proyecto `wiki-llm` en Go y dejar lista la fundación del MVP para generación de documentación Markdown a partir de OpenAPI.

---

## Contexto del producto

`wiki-llm` automatiza documentación técnica de APIs en Markdown compatible con Wiki (Azure DevOps u otros), usando fuentes como OpenAPI, Postman y código fuente.

Regla de diseño principal:

- La verdad técnica la determina el sistema.
- El LLM solo redacta/comunica en fases futuras.

Por lo tanto, en esta etapa:

- No se integra ningún LLM.
- Solo se prepara arquitectura y CLI base.

---

## Visión de ejecución (referencia futura)

Comando objetivo de uso:

```bash
wiki-llm generate api \
  --source ./input/openapi/api-cif.yaml \
  --output ./output/api-cif
```

Flujo esperado (no implementar ahora):

1. Leer fuente OpenAPI.
2. Extraer endpoints.
3. Normalizar a modelo interno.
4. Renderizar Markdown.
5. Escribir archivos (`index.md`, `apis/<nombre>.md`).

---

## Alcance de esta iteración (solo bootstrap)

Debes implementar únicamente:

1. Estructura base del proyecto en Go.
2. CLI mínima con comando raíz y subcomando `generate` placeholder.
3. Paquetes internos como módulos vacíos (placeholders).
4. `README.md` inicial con instrucciones básicas de ejecución.

No implementar en esta iteración:

- Parser de OpenAPI.
- Generación real de Markdown.
- Integración de LLM.
- Lógica de negocio real.

## Requisitos técnicos

1. Inicializar módulo Go.
2. Crear `main.go` con:
   - punto de entrada,
   - logging básico,
   - CLI mínima con subcomando `generate`.
3. El comando `wiki-llm generate` debe responder con mensaje claro tipo `not implemented yet` (placeholder válido).
4. Mantener implementación simple, legible y extensible.
